# Contributing
This project is free and open source software. Contributions and suggestions submitted in earnest are welcomed and encouraged.

## Submitting Issues: Bugs, Features and Documentation
1. Search the repository's open issues for the problem or feature you are reporting or requestion.
2. If you're unable to find a related open issue, open a new one using one of the provided templates. Be sure to include a title and clear description, as much  relevant information as possible.

## Standard Pull Request Model
This project uses the GitHub [Pull Request
(PR)](https://docs.github.com/en/github/collaborating-with-issues-and-pull-requests/about-pull-requests)
development workflow.

* For trivial chnges -- like fixing a spelling mistake or an off-by-one
error -- simply create a PR.
* For non-trivial changes, we encourage you to first post about your plans in an issue
(an existing one or a new one you create) so you can get useful design feedback or
suggestions before you start coding.  
* **All PRs should reference an open issue.**

## Style gude
### Coding style
I am a professional web developer and I have chosen to make this as low-tech as possible, so that those who aren't professional web developers can more easily contribute.
If you submit a Pull Request with code changes, please remember to keep the HTML, JavaScript and CSS stylesheet as simple as possible.  That means:
* PRs that pull in an external stylesheet will probably be rejected.
* PRs that add a JavaScript package from a CDN will probably be rejected.

## Language style
* Please run and heed your spellchecker and grammar checker.
* I'm happy to accept translations also.
* Be concise.
* Be helpful.
